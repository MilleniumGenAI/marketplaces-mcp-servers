# Marketplaces MCP Servers (Refactored)

Набор MCP (Model Context Protocol) серверов для поиска товаров на крупнейших российских маркетплейсах: **Wildberries**, **Ozon** и **Яндекс Маркет**. Интегрируется напрямую с ИИ-агентами (Claude Desktop, Cursor и др.) через платформу Apify.

## Особенности архитектуры
*   **Core-структура:** Общая логика (кэширование, троттлинг запросов, работа с Apify API) вынесена в ядро для стабильности.
*   **LRU Cache:** Защита от утечек памяти — кэш имеет лимит на количество записей и время жизни (TTL).
*   **Retry Mechanism:** Автоматические повторы запросов с экспоненциальной задержкой при ошибках сети или лимитах (429).
*   **Compact Mode:** Оптимизирован для ИИ-агентов — возвращает только ТОП-3 товара, экономя токены в контекстном окне.

## Быстрый старт

### 1. Предварительные требования
*   Python 3.10+
*   Аккаунт на [Apify](https://apify.com/) и API Token.

### 2. Настройка окружения
Создайте файл `.env` в корневой папке:
```env
APIFY_TOKEN=ваш_токен_apify
# Необязательные настройки
SEARCH_CACHE_TTL_SEC=900
MAX_CONCURRENT_SEARCHES=2
```

### 3. Запуск серверов
Для каждого маркетплейса предусмотрен отдельный сервер:
*   `python wb_mcp_server.py` — Wildberries
*   `python ozon_mcp_server.py` — Ozon
*   `python ym_mcp_server.py` — Яндекс Маркет

## Доступные инструменты (Tools)

### `search_wb` / `search_ozon` / `search_ym`
Основной инструмент поиска.
*   **Параметры:**
    *   `query` (string, required): Поисковый запрос или ID товара.
    *   `limit` (number): Максимум результатов (hard cap: 10).
    *   `compact` (boolean, default: true): Если `true`, возвращает только `top_products` (оптимально для ИИ).
*   **Логика WB:** Использует двухэтапный поиск (Google Search Scraper -> Card Parser) для обхода блокировок.

### `get_[name]_runtime_config`
Возвращает текущие настройки сервера (ID акторов, прокси, лимиты).

### `get_[name]_provider_status`
Возвращает метрики работы: количество успешных/неудачных запросов, латентность и статус токена.

## Для ИИ-агентов (Системный промпт)
Используйте эти инструменты, когда пользователь просит найти товары, сравнить цены или проверить наличие. 
*   **Совет:** Всегда начинайте поиск с `compact: true`.
*   **Кэширование:** Если вы повторно запрашиваете тот же товар в течение 15 минут, данные придут мгновенно из локального кэша.
