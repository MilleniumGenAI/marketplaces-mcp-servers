# Marketplaces MCP Servers

Набор MCP (Model Context Protocol) серверов для поиска товаров на крупнейших RUS маркетплейсах: **Wildberries**, **Ozon** и **Яндекс Маркет**. Интегрируется напрямую с ИИ-агентами (Claude Desktop, Cursor, OpenClaw и другие) через платформу Apify.

## Особенности архитектуры

* **Core-структура:** Общая логика (кэширование, троттлинг запросов, работа с Apify API) вынесена в ядро для стабильности.
* **LRU Cache:** Защита от утечек памяти — кэш имеет лимит на количество записей и время жизни (TTL).
* **Retry Mechanism:** Автоматические повторы запросов с экспоненциальной задержкой при ошибках сети или лимитах (429).
* **Compact Mode:** Оптимизирован для ИИ-агентов — возвращает только ТОП-3 товара, экономя токены в контекстном окне.

## Настройка окружения

Создайте файл `.env` в корневой папке и добавьте следующие переменные:

| Переменная | Описание | Значение по умолчанию |
| :--- | :--- | :--- |
| `APIFY_TOKEN` | **Обязательно.** Ваш API ключ от платформы Apify. | - |
| `MAX_CONCURRENT_SEARCHES` | Лимит параллельных запросов к Apify. Рекомендуется начать с `1` и увеличивать только если ваш тариф Apify позволяет больше параллельных запусков (Parallel Runs). | `1` |
| `SEARCH_CACHE_TTL_SEC` | Время жизни результата поиска в секундах (после этого запрос к Apify будет повторен). | `900` (15 мин) |
| `WB_APIFY_TIMEOUT_SEC` | Максимальное время ожидания ответа от актора (в секундах). | `120` |
| `WB_APIFY_RETRIES` | Количество попыток повторного запроса при сетевых сбоях. | `3` |

## Запуск серверов

Для каждого маркетплейса предусмотрен отдельный файл запуска:

* `python wb_mcp_server.py` — Wildberries (двухэтапный поиск)
* `python ozon_mcp_server.py` — Ozon (быстрый поиск через scraper)
* `python ym_mcp_server.py` — Яндекс Маркет (поиск через actor)

## Доступные инструменты (Tools)

### `search_wb` / `search_ozon` / `search_ym`

Основной инструмент поиска товаров. ИИ-агент вызывает его автоматически.

* **Параметры:**
  * `query` (string, required): Текст запроса (например, "айфон 17 Pro Max 1tb") или артикул.
  * `limit` (number): Максимальное количество товаров (максимум 10).
  * `compact` (boolean): Рекомендуется `true`. Возвращает краткую сводку по лучшим ценам.
  * `fresh_only` (boolean): Если `true`, игнорирует кэш и всегда делает новый запрос.

### `get_[name]_runtime_config`

Позволяет узнать, с какими ID акторов и настройками прокси работает сервер в данный момент.

### `get_[name]_provider_status`

Позволяет проверить "здоровье" сервера: статистику ошибок, количество успешных вызовов и время последней активности.
